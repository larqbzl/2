Сначала почини кнопку Download у SerpApi результатов. Сейчас при клике открывается страница с картинкой на сайте-источнике вместо скачивания файла на компьютер. Сделай так, чтобы сервер сам скачивал файл с original image_url, используя fetch на сервере, и отдавал его браузеру с заголовком Content-Disposition: attachment, чтобы браузер сразу сохранял файл на диск, а не открывал его как страницу. Эндпоинт может быть GET /api/download?url=... который проксирует файл через сервер. Примени это для всех источников, не только SerpApi, если у остальных та же проблема. Протестируй сам через curl, что эндпоинт отдаёт правильный Content-Type и Content-Disposition, и подтверди явным тестом в интерфейсе, что файл реально сохраняется в Downloads, а не открывает вкладку.

После того как это починишь и подтвердишь тестом, переходи к следующему этапу. В Batch добавь текстовое поле, куда можно вставить список поисковых запросов построчно, и для каждой строки отдельное числовое поле "сколько картинок нужно по этому запросу", например 20. Все строки должны обрабатываться параллельно, несколько запросов одновременно, а не по очереди одна за другой, чтобы десять запросов не занимали много времени. Результаты группируй по блокам, один блок на одну исходную строку-запрос, с заголовком (текст запроса) и количеством найденных картинок. Добавь индикатор прогресса вида "X из Y запросов обработано". Протестируй сам на пяти строках по десять-двадцать картинок каждая, покажи результат и время выполнения, прежде чем сказать что готово.

После подтверждения этого этапа сделай последний шаг — массовое скачивание. В результатах Batch добавь чекбоксы на каждой картинке и кнопку "выбрать все" на уровне каждого блока-запроса. Добавь кнопку Download selected, которая собирает все выбранные файлы в один zip-архив, используя тот же серверный проксирующий механизм скачивания, что мы только что чинили, чтобы не было проблемы с открытием страницы вместо скачивания файла. Внутри архива файлы должны быть разложены по подпапкам, одна подпапка на каждый запрос-блок, а имя каждого файла должно включать источник, часть текста запроса и порядковый номер. Протестируй сам, выбрав несколько картинок из разных блоков, скачай архив, открой его и проверь, что все файлы на месте, не битые, лежат в правильных папках и с понятными именами.
