Я хочу внести несколько важных изменений в проект Media Finder. Читай внимательно каждый пункт, не пропускай ничего. Не трогай логику фотографий — она работает нормально.

ЧАСТЬ 1 — BATCH-ВКЛАДКА: ДОБАВИТЬ ПОИСК И СКАЧИВАНИЕ ВИДЕО
Сейчас в Batch-вкладке работает только поиск фотографий. Нужно добавить точно такой же функционал для видео, но отдельно от фото.
Как должно работать:

Пользователь вводит список тем (каждая тема с новой строки), выбирает количество видео на каждую тему. По умолчанию стоит число 4.
После нажатия кнопки поиска — система ищет видео по каждой теме одновременно на двух источниках: Pixabay и Pexels. Оба источника уже подключены в проекте, используй существующие API-ключи и методы подключения.
Результаты отображаются карточками — на каждой карточке: превью видео (thumbnail), название, источник (Pixabay или Pexels), длительность, разрешение, и кнопка "Скачать".
Под всеми результатами должна быть одна большая кнопка "Скачать всё" — она скачивает все найденные видео разом одним кликом.

Качество скачивания — это критически важно:

Все видео скачивать строго в 1080p.
Если видео на Pixabay или Pexels доступно в 4K или 2K — это автоматически означает, что у него есть версия в 1080p. Нужно находить и скачивать именно версию 1080p, не выше.
720p использовать только как крайний запасной вариант, если версия 1080p реально отсутствует в API-ответе.
Никогда не скачивать 4K и 2K — только 1080p или ниже.

Для Pexels: при получении видео из API в объекте video_files есть массив с разными качествами. Выбирать файл, у которого width = 1920 (то есть 1080p). Если такого нет — брать ближайшее меньшее.
Для Pixabay: в API-ответе есть поле videos, внутри которого есть large (1080p), medium (640p), small (240p). Всегда брать large, это и есть 1080p.
Скачивание должно происходить напрямую через fetch и blob, без открытия сторонних страниц в браузере. Файл должен сохраняться на компьютер пользователя с нормальным именем файла (например: тема_источник_номер.mp4).

ЧАСТЬ 2 — SEARCH-ВКЛАДКА: ИСПРАВИТЬ СКАЧИВАНИЕ ВИДЕО
Сейчас когда пользователь нажимает кнопку Download на видео в Search-вкладке — браузер открывает новую вкладку со страницей на сайте Pixabay или Pexels, где нужно вручную снова нажимать скачать. Это неправильно и неудобно.
Исправь так: кнопка Download должна сразу скачивать видеофайл напрямую на компьютер пользователя без открытия каких-либо страниц. Качество — строго 1080p по той же логике, что описана выше в Части 1. Используй fetch + blob + искусственный клик по ссылке для прямого скачивания файла.

ЧАСТЬ 3 — SEARCH-ВКЛАДКА: ИСПРАВИТЬ ПРЕВЬЮ ВИДЕО С PIXABAY
Сейчас карточки видео с Pixabay в Search-вкладке отображаются без превью — просто чёрный прямоугольник с кнопкой воспроизведения. Карточки с Pexels отображают превью нормально.
Исправь отображение превью для Pixabay. В API-ответе Pixabay для видео есть поле userImageURL или поле picture_id, из которого можно собрать URL превью. Также можно использовать поле videos.large.thumbnail если оно есть. Найди в API-ответе Pixabay подходящее поле с thumbnail и используй его как src для тега img в карточке, так же как это сделано для Pexels.
