PART 1 — SEARCH LOGIC COMPLETE REWORK
═══════════════════════════════════════════════════

The app must find the 2-3 BEST results per source for each prompt.
Not random results — the BEST ones based on relevance and quality.

QUALITY REQUIREMENTS — STRICT:
- Videos: minimum 1920x1080 (1080p). If source has no 1080p — skip it.
- Photos: minimum 2000px on shortest side.
- Archive.org: pick the highest quality file available.
  Check file list at https://archive.org/metadata/IDENTIFIER
  and select the largest .mp4 or highest resolution file.
- Never show results below these thresholds.
- Show resolution badge on every single result card.

SOURCES AND HOW TO SEARCH EACH ONE:

── SOURCE 1: Archive.org ──
This is the most important source. Real car commercials,
TV reviews, test drives from 2000-2015.

Search URL:
https://archive.org/advancedsearch.php?q=QUERY+AND+mediatype:movies&fl[]=identifier,title,description,year,subject,downloads&sort[]=downloads+desc&rows=20&output=json

After getting results, for each item fetch its metadata:
https://archive.org/metadata/IDENTIFIER

From metadata extract:
- All available video files (.mp4, .avi, .mov)
- Pick the one with highest resolution / largest file size
- Show: title, year, file size, format, resolution if detectable
- Show download count from metadata (popularity indicator)

Each Archive.org card must have:
- Large thumbnail: https://archive.org/services/img/IDENTIFIER
- Title (full)
- Year in large bold orange
- File size and format
- Button "▶ Preview" — plays video inline in HTML5 player
- Button "⬇ Download Best Quality" — downloads the best file directly
- Button "🔗 Open Page" — opens archive.org/details/IDENTIFIER

Show maximum 3 best Archive.org results per search.
Best = highest download count + most relevant to query year range.

── SOURCE 2: Pexels ──
Search videos first, then photos.
Only return results 1920x1080 or higher for video.
Only return results 2000px+ for photos.

For video results show:
- Thumbnail with play button overlay
- Duration badge
- Resolution badge (e.g. "4K" or "1080p")
- Author name
- Button "▶ Preview" — plays inline
- Button "⬇ Download 1080p" — downloads 1080p version specifically

For photo results show:
- Full thumbnail
- Resolution (width x height)
- Author
- Button "⬇ Download Full Size"

Show maximum 3 best results from Pexels per search.

── SOURCE 3: Pixabay ──
Same rules as Pexels.
Videos: 1920x1080 minimum.
Photos: 2000px+ minimum.

Show maximum 3 best results from Pixabay per search.

── SOURCE 4: Wikimedia Commons ──
Search for car photos, interior shots, manufacturer press photos.
These are often real press photos of specific car models — very useful.

API: https://commons.wikimedia.org/w/api.php?action=query&list=search&srsearch=QUERY&srnamespace=6&srlimit=10&format=json&origin=*

For each result fetch image info:
https://commons.wikimedia.org/w/api.php?action=query&titles=PAGETITLE&prop=imageinfo&iiprop=url|size|mime&format=json&origin=*

Only show images 2000px+ minimum dimension.
Show: thumbnail, resolution, license type, author.
Button "⬇ Download Full Size"

Show maximum 3 best Wikimedia results per search.
